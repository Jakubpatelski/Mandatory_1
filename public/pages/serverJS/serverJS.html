1. JS Scope

Scoping in JavaScript refers to the set of rules that determine where variables and functions can be accessed within a program. JavaScript has two types of scope: global scope and local scope.

Global scope refers to variables or functions that are declared outside of any function, and can be accessed from anywhere in the program. Local scope, on the other hand, refers to variables or functions that are declared inside a function, and can only be accessed within that function.

In general, it is recommended to use const when declaring variables, as this keyword creates a variable that is read-only and cannot be reassigned a new value.

By using const and let appropriately, we can ensure that our variables are scoped correctly and that our code is more readable and maintainable. It also helps to prevent bugs caused by accidental reassignment of variables that should be read-only.

2. Callback Function

A callback function is a function that is passed as an argument to another function and is executed when a specific event occurs or when a certain portion of code is executed.

  // function
  function greet(name, callback) {
    console.log('Hi' + ' ' + name);
    callback();
  }
  
  // callback function
  function callMe() {
    console.log('I am callback function');
  }
  
  // passing function as an argument
  greet('Peter', callMe);
  
  // Output
  // Hi Peter
  // I am callback function

4. Node modules

Node modules- In Node.js, node_modulesis a directory that contains packages and dependencies installed by the Node Package Manager (NPM). When a developer installs a package or dependency using NPM, it is downloaded and stored in the node_modulesdirectory, along with any other dependencies required by that package. Node.js application requires a module or package, the require()function is used to load it. The use of node_modulesallows developers to easily manage and install dependencies for their Node.js projects. By using NPM to manage dependencies, developers can easily install, update, and remove packages as needed, without having to worry about managing dependencies manually

5. Strict Mode

Strict mode is a feature in JavaScript that enables a stricter set of rules for language syntax and behavior, making it easier to write more secure and reliable code. It is enabled by adding the statement 'use strict'; at the beginning of a script or function.

6. Functional Approach to Loops

A functional approach to loops is preferred because it avoids side effects, and involves using higher-order functions like map, filter, reduce, and forEach to manipulate collections of data without the need for traditional for or while loops.

Imperative programming is one that gives the computer a set of statements to perform a task.

Functional programming is a form of declarative programming. You tell the computer what you want done by calling a method or function.

let sum = 0;
for (let i = 0; i < nums.length; i++) {
   sum += nums[i];
}
const nums = [1, 2, 3, 4, 5];
const sum = nums.reduce((acc, curr) => acc + curr, 0);

In this example, reduce takes an initial value of 0 and an accumulator function that takes the current sum and the current element of the array and returns a new sum.

Functional programming encourages us to think of loops as transformations of data, rather than as imperative statements that mutate state. By using higher-order functions, we can write more declarative and composable code that is easier to reason about and less error-prone.
